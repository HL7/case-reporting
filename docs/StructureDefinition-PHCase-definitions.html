D.4.1.4 StructureDefinition: - Detailed Descriptions

Definitions for the StructureDefinition-PHCase Profile.

PHCase
Definition

For use in reporting suspected conditions from provider to public (population) health departments.

Control0..?
PHCase.id
Definition

unique id for the element within a resource (for internal references). This may be any string value that does not contain spaces.

Control0..1
Typestring
PHCase.extension
Definition

May be used to represent additional information that is not part of the basic definition of the element. In order to make the use of extensions safe and manageable, there is a strict set of governance applied to the definition and use of extensions. Though any implementer is allowed to define an extension, there is a set of requirements that SHALL be met as part of the definition of the extension.

Control0..*
TypeExtension
Alternate Namesextensions, user content
Comments

There can be no stigma associated with the use of extensions by any application, project, or standard - regardless of the institution or jurisdiction that uses or defines the extensions. The use of extensions is what allows the FHIR specification to retain a core level of simplicity for everyone.

PHCase.identifier
Definition

Identifiers assigned to this event performer or other systems.

NoteThis is a business identifer, not a resource identifier (see discussion)
Control0..*
TypeIdentifier
Requirements

Allows identification of the event as it is known by various participating systems and in a way that remains consistent across servers.

PHCase.definition
Definition

A protocol, guideline, orderset or other definition that was adhered to in whole or in part by this event.

Control0..*
TypeReference(ActivityDefinition)
Comments

[The allowed reference resources may be adjusted as appropriate for the event resource].

PHCase.partOf
Definition

A larger event of which this particular event is a component or step.

Control0..*
TypeReference(http://hl7.org/fhir/StructureDefinition/Any)
Requirements

E.g. Drug administration as part of a procedure, procedure as part of observation, etc.

Alternate Namescontainer
Comments

Not to be used to link an event to an Encounter - use Event.context for that.

PHCase.status
Definition

The current state of the event.

Control1..1
Binding​Codes identifying the stage lifecycle stage of the case
The codes SHALL be taken from PH Case Status Codes
Typecode
Is Modifiertrue
Comments

A nominal state-transition diagram can be found in the [[event.html#statemachine | Event pattern]] documentation

Unknown does not represent "other" - one of the defined statuses must apply. Unknown is used when the authoring system is not sure what the current status is.

PHCase.category
Definition

A code that classifies the kind of case report this is.clinical discipline, department or diagnostic service that created the report (e.g. infectious disease, poisoning). This is used for searching, sorting and display purposes.

Control0..1
BindingCodes indicating what triggered this case report. Based upon Lab Order Code, Lab Result Code, Lab Results and Diagnosis.
The codes SHOULD be taken from PH Case Category Codes
TypeCodeableConcept
Alternate Nameskind, type
PHCase.code
Definition

Reportable condition codes that caused this report to be generated.

Control0..1
BindingCodes indicating what triggered this case report. Based upon Lab Order Code, Lab Result Code, Lab Results and Diagnosis.
The codes SHOULD be taken from Case Report Codes
TypeCodeableConcept
PHCase.subject
Definition

The individual or set of individuals the case report is about.

Control0..1
TypeChoice of: Reference(Patient), Reference(Group)
Requirements

Links the event to the Patient context.

Alternate Namespatient
Comments

[For resources that aren't patient-specific, the set of allowed resources may be extended to include other things. Group should generally be retained unless there's certainty this resource won't be used for veterinary, research or public health settings where Group may be necessary (e.g. this cage of rats/crate of chickens, group of people in a 5 mile radious of the incident, etc.)].

PHCase.context
Definition

The encounter or episode of care that establishes the context for this event.

Control0..1
TypeChoice of: Reference(Encounter), Reference(EpisodeOfCare)
Requirements

Links the request to the Encounter context.

Alternate Namesencounter
Comments

This will typically be the encounter the event occurred within, but some events may be initiated prior to or after the official completion of an encounter or episode but still be tied to the context of the encounter or episode (e.g. pre-admission lab tests).

PHCase.issued
Definition

The date and time that this version of the report was made available to providers. For example the date on which the reporting party completed the report.

Control0..1
Typeinstant
Comments

May be different from the update time of the resource itself, because that is the status of the record (potentially a secondary copy), not the actual release time of the report.
